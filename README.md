# 3. RESTful API

## 3.1. Notice

### 3.1.1. Create

**Request**

- URL : api/notice

- Method : post

- Body : 
     
    ```json
    {
        "noticeTitle" : "공지제목",
        "noticeContent" : "공지내용"    
    }
    ```

### 3.1.2. Read


#### 3.1.2.1. 단일 요청

**Request**

- URL : api/notice/{공지 번호}

- Method : get

**Response**

단일 Notice 표현 Json 객체.


#### 3.1.2.2. 페이지 단위 요청

**Request**

- URL : api/notice/page/{페이지 번호}

- Method : get

**Response**

단일 페이지 내의 Notice 리스트 표현 Json 객체.

### 3.1.2.3. 조건 검색 요청

**Request**

- URL : api/notice/page/{페이지 번호}?opt={검색옵션}&key={검색어}

    검색옵션으로는 noticeTitle, noticeContent, noticeRegDate가 가능 검색옵션은 검색 대상이되는 공지사항의 속성을 가리킨다.

- Method : get

**Response**

{검색옵션} 속성에 {검색어}를 포함하는 공지사항을 검색한 결과의 {페이지 번호}페이지.

### 3.1.3. Update

**Request**

- URL : api/notice/{공지 번호}

- Method : patch

- Body : 
     
    ```json
    {
        "noticeTitle" : "공지제목",
        "noticeContent" : "공지내용"    
    }
    ```

### 3.1.4. Delete


**Request**

- URL : api/notice/{공지 번호}

- Method : delete
